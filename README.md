# Gaeuly - Portofolio
- This is the frontend interface for Modern site Portoflio. Built with React, Vite, and Tailwind CSS.


# Prerequisites
Before you begin, ensure you have met the following requirements:
- Node.js: v18.x or higher.
- Supabase
- NPM 

# Setup Supabase
- Change name file example.env -> .env
- Get your superbase URL and Anon key then enter it in .env

- Create a new table or you can use SQL Editor then copy and paste the text below
```bash
-- 1. Create Table Projects
create table projects (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  title text not null,
  slug text unique not null,
  description text not null,
  image text,
  technologies text[], -- Ini array string (ex: ['React', 'TS'])
  live_url text,
  github_url text,
  is_featured boolean default false
);

-- 2. Enable RLS (Security)
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;

-- 3. Open Access
CREATE POLICY "Public projects are viewable by everyone" ON projects FOR SELECT USING (true);
CREATE POLICY "Auth users can insert projects" ON projects FOR INSERT TO authenticated WITH CHECK (true);
CREATE POLICY "Auth users can update projects" ON projects FOR UPDATE TO authenticated USING (true);
CREATE POLICY "Auth users can delete projects" ON projects FOR DELETE TO authenticated USING (true);
```
- Create 1 more table for posts Or Copy&Paste This SQL
```bash
-- Create Posts Table
create table posts (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  updated_at timestamp with time zone default timezone('utc'::text, now()),
  title text not null,
  slug text unique not null,
  excerpt text,
  content text, -- Markdown content
  image text,   -- Cover Image URL
  is_published boolean default false
);

-- Enable Security (RLS)
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

-- Access Policies
CREATE POLICY "Public posts are viewable by everyone" ON posts FOR SELECT USING (true);
CREATE POLICY "Auth users can insert posts" ON posts FOR INSERT TO authenticated WITH CHECK (true);
CREATE POLICY "Auth users can update posts" ON posts FOR UPDATE TO authenticated USING (true);
CREATE POLICY "Auth users can delete posts" ON posts FOR DELETE TO authenticated USING (true);
```

# Create Account
- To log in to the dashboard to upload blogs and projects, you must first create an account.
```bash
- Go to supabase then go to the Authentication tab 
- Then click Add user -> Create new user 
- Then enter your email and password example: 
- gaeulyportfolio@gmail.com | password: admin
```
- If you have created an account, do the installation first. 
- Then go to https://your.domain.com/login

# Installation Guide
- Open your terminal and navigate to this frontend folder.
- Install the required dependencies:

```bash
npm install
```

# Running in Development
- To start the development server with hot-reload:

```bash
npm run dev
```
- Open your browser (or the port shown in terminal).


# Building for Production
- To deploy this frontend, you need to build it into static files (HTML/CSS/JS).

- Run the build command:
```bash
npm run build
```
- The output files will be in the dist/ folder.
- Upload the contents of the dist/ folder to your web hosting (Vercel, Netlify).
